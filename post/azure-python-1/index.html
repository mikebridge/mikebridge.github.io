Deploying Microservices on Azure With Kubernetes

 Posted on February 26, 2019  |  2 minutes  |  220 words  |  Mike Bridge

As we added more data-scientists to our team, we discovered that we had several problems integrating what they were producing with our existing software. We are comfortable in producing software in C# and deploying them on Azure, but most data science is done in Python, and integrating and managing these microservices required us to develop some new skills.

Some challenges we had are:

  • Data Scientists work in R or Python, but we primarily develop in C#
  • The kinds of services we are developing are best implemented as microservices, but managing infrastructure, deploying and is not something we want to have our data scientists to spend their time on

There are several solutions in Azure for deploying Python, including Azure Functions or as a App Service on Linux, or even to a Docker Container in an App Service. Ultimately we decided that this still required a lot of ongoing maintenance effort. Each time we deployed a new service, someone would have to set up a new HTTPS service, add it to the deployment, make it available in the various environments (Testing, Staging, QA, etc.). This is tedious, and frankly requires a different set of technologies from what we currently use to automate our C# and TypeScript environments.

So we decided to move to Kubernetes as a way of managing all these.
